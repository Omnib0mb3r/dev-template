## Feature Workflow

All feature and change work follows the **superpowers pipeline, augmented with OTLC layers**. Start here for any new feature, bug, or improvement.

### PRE — before brainstorming

1. Read this file in full (Vision, Architecture, Decisions Made, Open Questions).
2. **Vision check** — if `## Vision` is empty, define/confirm it with the user before brainstorming begins.
3. **Visual Companion** — if the topic involves layouts, mockups, designs, style choices, or flow diagrams, offer to launch the Visual Companion.

### RUN — superpowers pipeline

1. **Brainstorm** (`superpowers:brainstorming`) — Design the feature collaboratively. Produces a spec in `docs/superpowers/specs/`.
2. **Plan** (`superpowers:writing-plans`) — Auto-invoked at end of brainstorm. Produces a task-by-task plan in `docs/superpowers/plans/`.
3. **Execute** (`superpowers:subagent-driven-development`) — Agents implement the plan task-by-task with spec + quality review after each task.
4. **Finish** (`superpowers:finishing-a-development-branch`) — Merge, create PR, or discard. Auto-invoked at end of execution.

### POST — after the brainstorm spec is approved (BEFORE plan phase)

Update this file:
- **Decisions Made** — append a new dated entry. Check for conflicts before adding.
- **Open Questions** — append new questions; mark resolved with `[x]`, never delete.
- **Architecture** — update only if structural decisions changed.

### END — after the session ends

- **Session Log** — append a brief dated entry below.
